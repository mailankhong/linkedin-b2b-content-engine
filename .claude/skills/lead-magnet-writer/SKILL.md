# Lead Magnet Writer

Lead magnet posts are the highest-converting post type on LinkedIn — but only when the promise is specific and the ask is low. "DM me 'GUIDE' and I'll send you my playbook" works because it's frictionless. "Schedule a call to learn more" doesn't — that's a sales pitch disguised as generosity. The post must give away something genuinely useful, and the reader must believe they're getting value before any commitment.

This skill writes the post that promotes the resource. Uses 10 proven templates extracted from 40+ viral lead magnet posts, organized by psychological trigger.

**Pre-flight:** Run preflight.md. Client: [name provided by user].
**Required:** Voice Profile (post in client's voice), ICP (frame around audience pain points).

## Phase 2: Template Selection

### Template 1: Result + System Giveaway
Best for: Proven systems with clear ROI metrics
```

[Problem: What most people struggle with]
[Pain points: 3-4 bullets]
[Insight: Why the common approach fails]
[What's inside: 5-7 arrow bullets with specific deliverables]
[Social proof: 2-3 results]
[CTA: Connect + Comment keyword]
```

### Template 2: Unconventional Approach
Best for: Non-obvious strategies or channels
```
[Hook: Name the unconventional thing]

[CTA]
```

## Phase 3: Hook Generation

Lead magnet posts have their own hook logic — they need to sell the value of the resource immediately. Always generate **both option sets** and present them together so Mai Lan can choose:

### Option A: Lead Magnet-Specific Hooks (5 patterns)
These are tuned for giveaway/resource posts and tend to perform well for lead magnets specifically:

**Pattern A: Specific Result**
"I [built/created/documented] a [resource] that [does impressive thing]"

**Pattern B: Cost Replacement**
"NEVER [pay for/hire] X again. This [resource] does it for free."

**Pattern C: Speed Claim**
"This [resource] does in [short time] what takes most people [long time]."

**Pattern D: Investment Payoff**
"I spent [time/money] building [thing]. Giving it away for free."

**Pattern E: Bold Announcement**
"I just released a free [resource] that [bold outcome claim]."

### Option B: Hook Generator (broader range)
Run hook-generator for this topic to generate 10-15 additional variations across Desire / Curiosity / Fear triggers. These may surface angles the lead-magnet-specific patterns miss.

**Present both sets together.** Recommend the top 2-3 overall (mixing both options). Ask Mai Lan which to use before writing the full post.

## Phase 4: Write the Full Post

### Structure Requirements
- Hook: 1-3 lines with specific result or metric
- Problem: What most people struggle with (2-3 lines)
- Pain agitation: 3-4 specific pain points using ✗ or dash bullets
- Transition/insight: Why the common approach fails
- What's inside: 5-7 deliverables using → arrow bullets
- Social proof: 2-3 specific results (if available)
- CTA: 2-step process (Connect + Comment keyword)
- Optional: P.S. for repost priority

### Writing Style
- Short paragraphs (1-3 lines max)
- Heavy white space between sections
- Specific numbers everywhere
- Concrete timeframes
- Use → for deliverable lists
- Conversational tone, not corporate

### Deliverable Lines Must Be Specific
- BAD: "Tips for better content"
- GOOD: "The exact hook formula that generated 34 inbound leads from one post"

### Psychological Triggers
- **Specificity**: Exact numbers, timeframes, metrics
- **Contrast**: Before/after, most people vs you
- **Authority**: Years of experience, investment, scale
- **Simplicity**: "takes 10 minutes", "just paste and go"
- **Anti-guru**: "No fluff", "not theory", "the exact system"
- **Cost comparison**: What others charge for similar

## CTA Best Practices

```
Want [the thing]?

1. Connect with me (so I can DM you)
2. Comment "[KEYWORD]"

I'll send it to your DMs.
```

**Keyword rules:**
- Short and memorable (1 word, all caps)
- Related to the topic (HOOKS, SYSTEM, TOOLS, AUDIT, SKILLS)
- Easy to type on mobile

Optional: "PS: Repost for priority access ♻️"

## Verification

**Correctness:**
```
□ Hook includes a specific metric or bold claim — not a vague benefit statement? Yes → pass. No → rewrite the hook.
□ Deliverable lines describe specific outcomes ("the exact hook formula that generated 34 leads") — not vague benefits ("tips for better content")? Yes → pass. No → rewrite each vague deliverable.
□ CTA is a 2-step process: Connect + Comment keyword — not just "DM me"? Yes → pass. No → fix the CTA.
```

**Completeness:**
```
□ Post includes: hook, problem statement, pain points, what's inside (5–7 deliverables), CTA? Yes → pass. No → add missing sections.
□ Hook options presented from both lead-magnet-specific patterns AND hook-generator before selection? Yes → pass. No → generate both sets.
□ Keyword is short (1 word, all-caps), memorable, and related to the topic? Yes → pass. No → replace it.
```

**Context-fit:**
```
□ Post written in the client's documented voice — not generic giveaway post language? Yes → pass. No → apply the Voice Profile.
□ Resource promised in the post is the same resource that will be built in lead-magnet-builder? Yes → pass. No → align them before publishing.
□ Value is visible above the "see more" fold — someone can tell what they're getting without clicking? Yes → pass. No → front-load the value.
```

**Consequence:**
If this post is published and comments come in but the resource isn't ready: what is the reputational consequence?
→ Confirm the resource is built before publishing. If the answer is "resource not started yet" — do not publish the post.
