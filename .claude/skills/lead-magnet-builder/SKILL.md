# Lead Magnet Builder

The post creates a public promise. This skill builds the actual resource behind it. The most common failure: the resource is comprehensive but not useful. Nobody wants a 30-page guide. They want a 2-page checklist they can use today. The best lead magnets are tools, not textbooks — something the reader applies to their own situation within 10 minutes of opening it.

6 format options: playbooks, guides, templates, checklists, swipe files, system blueprints.

**Pre-flight:** Run preflight.md. Client: [name provided by user].
**Required:** Voice Profile, ICP (structure around audience pain points and desired outcomes).

## Quality Gate

Insufficient input → stop. State gap, impact, fix option. Never invent or fabricate.

1. **Input** - Understand what was promised in the lead magnet post

## Phase 2: Format Selection

### Format 1: The Playbook (Most common)
**Best for:** Systems, strategies, step-by-step processes
**Length:** 10-25 pages
**Structure:** Cover page + table of contents + chapters + action items
**When to use:** When you promised a "system", "playbook", "blueprint", or "guide"

### Format 2: The Checklist
**Best for:** Audits, reviews, optimization workflows
**Length:** 1-3 pages
**Structure:** Categorized yes/no or do/don't items with explanations
**When to use:** When you promised an "audit", "checklist", or "review system"

### Format 3: The Swipe File
**Best for:** Templates, examples, copy-paste resources
**Length:** 5-15 pages
**Structure:** Collection of ready-to-use examples organized by category
**When to use:** When you promised "templates", "hooks", "scripts", "examples"

### Format 4: The Template Pack
**Best for:** Fill-in-the-blank frameworks, SOPs, worksheets
**Length:** 3-10 pages per template
**Structure:** Instructions + blank template + filled example
**When to use:** When you promised "templates", "frameworks", "worksheets"

### Format 5: The System Blueprint
**Best for:** Technical setups, tool configurations, automation workflows
**Length:** 10-20 pages
**Structure:** Architecture overview + step-by-step setup + troubleshooting
**When to use:** When you promised a "system", "setup guide", "tech stack"

### Format 6: The Mini Course (Outline)
**Best for:** When video or multi-part delivery makes more sense
**Length:** Outline + scripts for 3-7 short lessons
**When to use:** When you promised a "course", "masterclass", "breakdown"

## Phase 3: Universal Resource Principles

1. **Deliver on EVERY promise from the post** — Go through the post bullet by bullet. Every arrow item must have a corresponding section.
2. **Lead with the most valuable content** — Put the most actionable content in the first section. Most people won't read everything.
3. **Make everything actionable** — Every section ends with something the reader can DO.
4. **Include examples** — For every framework or template, include at least one filled-in example.
5. **Be specific, not comprehensive** — A 5-page guide with real examples beats a 50-page guide full of generic advice.

## Phase 4: Content Generation Templates

### Playbook Structure

```
COVER PAGE: Title + Subtitle + Your name/brand

TABLE OF CONTENTS

INTRODUCTION (1 page):
- Who this is for
- What they'll get (specific outcomes)
- How to use it
- Quick win: One thing they can do in the next 5 minutes

SECTION 1: [Highest-value content first]
- Concept explanation (2-3 paragraphs max)
- Step-by-step process
- Real example (filled in, not blank)
- Action item: What to do right now

SECTIONS 2-5: [Remaining sections — same structure]

BONUS SECTION (optional):
- Advanced tips
- Common mistakes to avoid
- FAQ

CLOSING:
- Quick recap (3-5 bullets)
- What to do next (the ONE most important action)
- Soft CTA for your service (1-2 sentences)
```

### Checklist Structure

```
TITLE: [What this checks/audits]

INSTRUCTIONS: How to use, when to run, what to do with results.

CATEGORY 1: [Area being checked]
✓ [Item to check] - [Why it matters in 1 sentence]
✓ [Item to check] - [Why it matters]

SCORING: How to interpret results.

PRIORITY FIXES: If you can only fix 3 things, fix these.
```

### Swipe File Structure

```
TITLE: [Number] [Type of examples] for [Outcome]

HOW TO USE THIS FILE:
Don't copy word for word. Use as inspiration. Adapt structure and style to your voice.

CATEGORY 1: [Type]
Example 1: [Full example]
Why it works: [1-2 sentences]

Example 2: [Full example]
Why it works: [1-2 sentences]
[5-10 examples per category]
```

### Template Pack Structure

```
TEMPLATE: [Name]

PURPOSE: What this template is for.

INSTRUCTIONS:
1. [Step to fill in]
2. [Step to fill in]

BLANK TEMPLATE:
[Template with [BRACKETS] for fill-in sections]

FILLED EXAMPLE:
[Same template filled in with a real example]

TIPS:
- [Common mistake to avoid]
- [Pro tip for better results]
```

## Phase 5: Delivery Formatting

### For PDF/Document
- Clean, modern layout
- Consistent header styling
- Plenty of white space
- Page breaks between major sections
- Table of contents with page numbers

### For Notion
- One main page with toggles for each section
- "Start Here" section at the top
- Callout blocks for key takeaways

### Soft CTA Placement
Every resource should include a subtle CTA — not a sales pitch, a natural mention:
- Brief mention in the introduction ("I do this for clients every day")
- Brief mention at the end ("If you want this done for you, [one sentence]")
- In the author bio section

Tone: Helpful, not pushy. The reader should feel they got massive value for free.

## Quality Checklist

- ✅ Every deliverable from the post is addressed in the resource
- ✅ First section delivers immediate value (the "quick win")
- ✅ Every framework includes a filled-in example
- ✅ Action items are clear and specific
- ✅ Formatted for easy scanning (not walls of text)
- ✅ Reader can get value in under 10 minutes
- ✅ CTA is present but subtle
- ✅ You'd actually want to read this if someone sent it to you
- ✅ Specific to the niche, not generic advice repackaged

## Verification

**Correctness:**
```
□ Every deliverable listed in the lead magnet post has a corresponding section in the resource? Yes → pass. No → add the missing section.
□ First section delivers the highest-value content — not background or context? Yes → pass. No → reorder so the most actionable content comes first.
□ Every framework or template includes a filled-in example — not just a blank template? Yes → pass. No → fill one in.
```

**Completeness:**
```
□ Resource format matches what was promised in the post (playbook/checklist/swipe file/template/blueprint/mini-course)? Yes → pass. No → rebuild in the correct format.
□ Every section ends with an action item the reader can do immediately? Yes → pass. No → add action items.
□ Soft CTA present (brief, not pushy) in introduction and closing? Yes → pass. No → add it.
```

**Context-fit:**
```
□ Resource written in the client's voice — not generic "content marketing" tone? Yes → pass. No → apply the Voice Profile.
□ Resource is specific to the client's niche — could not be published by a competitor without significant changes? Yes → pass. No → add client-specific examples and data.
□ Reader can extract value in under 10 minutes — not a 50-page generic guide? Yes → pass. No → cut to the highest-value content.
```

**Consequence:**
If a person receives this resource after commenting on the lead magnet post: what is the most likely disappointment?
→ Answer before finalizing. If the answer is "the resource doesn't deliver on what the post promised" or "no actionable quick win in the first section" — fix it before the post goes live.
