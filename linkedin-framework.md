# LinkedIn Framework

Platform rules for B2B LinkedIn content. Load at the start of every copy-developer session.

---

## 1. Hook Patterns — 5 B2B-Specific Formulas

Use these as starting points. Select based on what the content can actually support.

---

**Formula 1 — The Specific Result**
Structure: `[Concrete metric] in [timeframe]. Here's what we did:`
Example: "14 qualified calls from one post. No ads. Here's the framework:"

---

**Formula 2 — The Pattern Break**
Structure: `Everyone in [industry] does [X]. The [N]% who don't do [Y] instead.`
Example: "Everyone in B2B SaaS runs discovery calls the same way. The 8% who close at 40%+ don't."

---

**Formula 3 — The Named Mistake**
Structure: `[Specific audience] loses [specific outcome] because of [specific mistake]. Here's why:`
Example: "B2B founders lose their best pipeline in Q4 because of one follow-up timing mistake. Here's why it happens:"

---

**Formula 4 — The Contrarian Observation**
Structure: `I've talked to [N] [audience type]. The assumption everyone shares is wrong.`
Example: "I've talked to 200 sales leaders this year. The assumption every single one of them shares is wrong."

---

**Formula 5 — The Earned Claim**
Structure: `[Specific credential or experience]. The [number] thing I wish I knew earlier:`
Example: "6 years building B2B pipelines from scratch. The one thing I'd tell every founder:"

## 2. Formatting Rules — Non-Negotiable

Apply to every post. The only override is a documented `formatting-profile.md` that says otherwise.

**Short sentences.** One idea per sentence. Impact lines: under 12 words. Context-setting lines can be longer — no run-ons.

**White space.** Break after every 2–3 lines. LinkedIn is skimmed. Paragraphs over 3 lines lose readers.

**Numbers, not words.** "7 clients" not "several clients." "3 months" not "a few months." "47% of founders" not "most founders."

**No em dashes.** Em dashes read as AI. Use a period. Start a new line. Never use an em dash as a substitute for either.

**Active voice.** "We closed the deal" not "the deal was closed." "I made the call" not "the call was made."

**No jargon.** If your ICP wouldn't say it over coffee, cut it.

**No hashtags in the body.** Three maximum, at the very end only. Voice profile says no hashtags → use none.

**External links never in the post body.** LinkedIn suppresses reach on posts with outbound links. Note at draft end: "Link goes in first comment."

## 3. CTA Guide

**Engagement CTAs** — ask for opinions, experiences, or saves. Use when goal is reach, trust-building, or community.
- "What's been your experience with this?"
- "Which of these applies to you right now?"
- "Save this for [specific situation]."
- "If this sounds familiar, I'd like to know why."

**Promotional CTAs** — direct the reader to an action. Use only when the post has delivered enough value to earn the ask.
- "DM me [specific word] and I'll send you [specific resource]."
- "Comment [word] below and I'll follow up."
- "If you're dealing with [specific situation], let's have a 20-minute call."
- "Book a call if [specific ICP situation] describes you."

**Ratio:** 60–70% engagement CTAs, 30–40% promotional CTAs.

| Category | Default CTA type |
|----------|-----------------:|
| Teach | Engagement (invite application or resource request) |
| Lead | Engagement (invite disagreement or extension) |
| Prove | Promotional (invite similar situation or direct conversation) |
| Connect | Engagement only — no promotional CTA |
| Respond | Engagement or none — hard CTA is rarely earned |

**CTA rules:**
- One CTA per post. Never two.
- Never end with: "What do you think?" / "Thoughts?" / "Drop a comment below."
- Never end with a summary that restates the post.
- CTA must be earned by the content. If the post hasn't demonstrated value, the ask fails.

## 4. AI-Tell Red Flags

Remove every instance. Not reduce — remove.

**Vocabulary (never use):**
in today's landscape / in today's fast-paced world / it's important to note / as we navigate / the reality is / here's the thing / let me be honest / make no mistake / let that sink in / read that again / full stop / game-changer / next-level / deep dive / that said / leverage (as a verb) / utilize / synergies / circle back / in conclusion / it's worth noting / key takeaways / I wanted to share / I'm excited to announce (unless genuinely excited) / groundbreaking / revolutionary / transformative / cutting-edge / robust / scalable / best practices / touch base / actionable insights / move the needle / think outside the box

**Structural patterns — flag and rewrite:**
- Em dashes used for dramatic effect or as a substitute for a period
- Every paragraph the same length — real writing has variation
- Overly perfect parallel structure: three bullets that all begin and end the same way, with identical rhythm
- Questions used as the opening hook
- Posts that open with "I" as the very first word
- Conclusions that restate everything just covered

## 5. Self-Editing Checklist

Run before every delivery. Fix every failed item.

**Hook:**
```
□ First line makes a specific claim, not a vague one? Yes → pass. No → rewrite.
□ Earns the "see more" click without asking a question? Yes → pass. No → rewrite.
□ Avoids starting with "I"? Yes → pass. No → rewrite.
□ Would stop your scroll in a crowded feed? Yes → pass. No → rewrite.
```

**Body:**
```
□ One idea per line? Yes → pass. No → break up.
□ No paragraph over 3 lines? Yes → pass. No → split.
□ Every claim that has a number uses the number? Yes → pass. No → add it.
□ Zero em dashes? Yes → pass. No → replace each with a period or line break.
□ Active voice throughout? Yes → pass. No → rewrite passive sentences.
□ Zero AI vocabulary from the list above? Yes → pass. No → remove every instance.
□ Each section follows logically from the one before? Yes → pass. No → reorder.
```

**Style:**
```
□ Sounds like a specific person, not polished-generic content? Yes → pass. No → find what's missing from the Voice Profile.
□ Zero banned words from the client's Voice Profile? Yes → pass. No → remove all of them.
□ Swap test: could a direct competitor post this exact draft? Yes → make it more specific. No → pass.
```

**CTA:**
```
□ One CTA only? Yes → pass. No → cut to one.
□ CTA type appropriate for category? (Connect = engagement only. Respond = soft or none.) Yes → pass. No → fix.
□ Avoids "What do you think?" / "Thoughts?" / "Drop a comment below"? Yes → pass. No → rewrite.
□ Post earned the ask? Yes → pass. No → add more value or soften the CTA.
```

## 6. Performance Benchmarks

Use to calibrate expectations and evaluate post performance after publishing.

**Engagement rate targets:**
- 3–5%: solid. Post is resonating with the existing audience.
- 5–8%: strong. Post is hitting a real nerve.
- 8–10%: excellent. Candidate to repurpose or build a series from.
- 10%+: viral for this niche. Study the structure and repeat it.

*(Engagement rate = (reactions + comments + shares) / impressions × 100)*

**Signal definitions:**

| Signal | What it means |
|--------|---------------|
| Meaningful comments | Comments that add a perspective, share an experience, or ask a follow-up question. Emoji reactions and "great post!" do not count. |
| Saves | Underrated signal. High saves indicate reference value. Strongest indicator for Teach posts. |
| Shares / reposts | Highest-value signal for Lead posts. Indicates the idea spread beyond the existing network. |
| Connection requests from a post | Best pipeline signal. The person read this and decided they wanted more access to this person. |

**Post type benchmarks (relative):**
- Teach posts: high saves, moderate comments
- Lead posts: high shares and comments, lower saves
- Prove posts: lower reach, higher DMs and direct pipeline signals
- Connect posts: high comments, high follows — lower saves
- Respond posts: depends entirely on timing; strong if published in first 12 hours of an event cycle

## Verification

**Correctness:**
```
□ All 8 formatting rules applied in the draft (short sentences, white space, numbers, no em dashes, active voice, no jargon, hashtags at end, no links in body)? Yes → pass. No → fix each violation.
□ Zero AI-tell vocabulary from Section 4? Yes → pass. No → remove every instance.
□ CTA matches the category default from Section 3? Yes → pass. No → fix.
```

**Completeness:**
```
□ linkedin-framework.md loaded at session start (before any drafting)? Yes → pass. No → load it now and re-check the draft against all rules.
□ Self-editing checklist (Section 5) run before delivery? Yes → pass. No → run it.
□ Hook formula used is from Section 1 or hook-generator Phase 2? Yes → pass. No → replace with a formula-based hook.
```

**Context-fit:**
```
□ Formatting-profile.md checked — if it exists and overrides any default here, the override is applied? Yes → pass. No → check the client folder.
□ Benchmark context correct for this post type? (Prove posts expect DMs, not saves. Teach posts expect saves, not DMs.) Yes → pass. No → adjust expectations.
□ Hook formula matches the category of the post? (Prove posts use Formula 1 or Metric Drop. Lead posts use Formula 4 or Contrarian.) Yes → pass. No → re-select the formula.
```

**Consequence:**
If this post is published without checking against this framework: which rule is most likely to have been violated?
→ Name it before delivering. If the answer is "em dash is present" or "the CTA is a summary restatement" — fix it now.
